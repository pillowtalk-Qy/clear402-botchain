# BOT Chain Integration

## Summary

Clear402 for BOT Chain uses BOT Chain testnet as the EVM settlement layer for a
guarded AI-agent payment demo. The local x402-style provider requests payment,
Clear402 builds and checks a `PaymentContext`, and the EVM settlement proof is
recorded through `ServiceEscrow` transactions on BOT Chain testnet.

## Why BOT Chain Fits

AI agents may issue many small paid API calls. That requires a settlement layer
that is fast, low-cost, EVM-compatible, and easy for existing Solidity tooling.
BOT Chain fits this demo because it can host the escrow contract while Clear402
adds the payment safety layer around the transaction.

## Settlement Contract

`contracts/ServiceEscrow.sol` is intentionally minimal:

- `fund(bytes32 paymentContextHash, address provider, uint256 amount)`
- `deliver(bytes32 paymentContextHash)`
- `refund(bytes32 paymentContextHash)`

The key binding is `paymentContextHash`. Clear402 computes this hash from the
paid resource, provider, quote, nonce, chain, amount, and guard policy. The
escrow transaction must reference the same hash that passed the guard pipeline.

## Evidence Requirements

The BOT Chain evidence pack must include:

- contract address;
- deployment transaction hash;
- at least one `fund(...)` transaction hash;
- preferably one `deliver(...)` or `refund(...)` transaction hash;
- explorer links;
- `paymentContextHash`;
- action status and block number;
- a statement that this is BOT Chain testnet evidence.

Amounts in x402-style challenges and `PaymentContext` are recorded as decimal
digit strings in base units. For the demo, `1000000000000` means `0.000001 BOT`
when interpreted with 18 decimals.

Evidence generated by the scripts is written to `evidence/botchain/`.

## Local Commands

```bash
pnpm contracts:compile
pnpm botchain:deploy:escrow
pnpm botchain:escrow fund
pnpm botchain:escrow deliver
pnpm botchain:escrow read
```

## Demo Flow

1. Start runtime, provider, and dashboard.
2. Create an agent payment mission.
3. Inspect the x402-style 402 challenge.
4. Run the guard pipeline.
5. Show the `PaymentContext` and `paymentContextHash`.
6. Execute a tiny BOT Chain escrow interaction.
7. Open the transaction in the BOT Chain explorer.
8. Export evidence.
9. Run Attack Lab to show unsafe flows are blocked.

## Safe Wording

Use:

> Clear402 binds x402-style agent payments to BOT Chain testnet escrow evidence.

Avoid:

> Clear402 is a production payment gateway.

Avoid:

> Every x402 payment is settled on BOT Chain.

Avoid any wording that implies mainnet-ready custody infrastructure.
